# UniPilot — الجودة والامتثال (Quality & Compliance)

هل المشروع خاضع لمعايير الجودة العالمية ومعايير ISO؟

---

## 1. توضيح مهم

**UniPilot غير حاصل على شهادة رسمية من أي جهة معتمدة لـ ISO** (مثل ISO 9001 أو ISO 27001). الحصول على الشهادة يتطلب تدقيقاً من جهة إصدار شهادات معتمدة (certification body) ولا يمكن منحها من خلال التوثيق وحده.

ما يلي يوضح **مدى توافق المشروع** مع ممارسات ومعايير الجودة والأمان، وما الذي يجهّز المشتري للسير نحو الشهادة لاحقاً إذا رغب.

---

## 2. معايير ISO ذات الصلة وتوافق المشروع

### ISO 9001 (نظام إدارة الجودة — Quality Management)

| متطلب / ممارسة | وضع المشروع | ملاحظات |
|----------------|-------------|----------|
| توثيق العمليات والإجراءات | ✅ جزئي | RUNBOOK، ADMIN_GUIDE، DEPLOYMENT_READINESS، MIGRATIONS، BACKUP_RESTORE |
| تحديد متطلبات المنتج | ✅ | PRODUCT_SPEC، PREMIUM_PRODUCT_READINESS |
| مراقبة المنتج (اختبارات) | ✅ | وحدة (Vitest)، تكامل API (supertest)، E2E (Playwright)، CI |
| تحسين مستمر | ⚠️ | لا توجد دورة PDCA موثقة رسمياً؛ المشتري يمكنه إضافتها |
| سجلات ووثائق قابلة للتتبع | ✅ | توثيق الإصدار، تغييرات Schema في MIGRATIONS |

**الخلاصة:** المشروع يوفّر أساساً جيداً من التوثيق والاختبار؛ للحصول على شهادة ISO 9001 يلزم بناء نظام إدارة جودة كامل (سياسات، إجراءات، سجلات، مراجعات إدارية) لدى المشتري.

---

### ISO/IEC 27001 (أمن المعلومات — Information Security)

| ضابط / ممارسة | وضع المشروع | ملاحظات |
|----------------|-------------|----------|
| أمان التطبيق (OWASP-style) | ✅ | Helmet، rate limiting، CORS، تحقق المدخلات (Zod)، استعلامات مُعلّمة (لا SQL injection) |
| إدارة الهوية والوصول | ✅ | JWT، bcrypt، عدم تخزين كلمات مرور نصية، سر إنتاج قوي (JWT_SECRET) |
| حماية البيانات | ✅ | تصدير/حذف بيانات، سياسة خصوصية، شروط خدمة، تسجيل الموافقة (terms_accepted_at) |
| أمان البنية | ✅ | HTTPS (عبر المنصة)، أسرار عبر متغيرات بيئة، SECURITY.md، SECURITY_UPDATES_POLICY |
| الاستجابة للحوادث | ⚠️ | DISASTER_RECOVERY، BACKUP_RESTORE؛ لا خطة استجابة حوادث رسمية موثقة |
| تقييم المخاطر | ❌ | لا يوجد تقييم مخاطر أمنية رسمي (يُجرى عادة عند بناء ISMS للشهادة) |

**الخلاصة:** المشروع متوافق مع ممارسات أمنية شائعة (OWASP، headers، rate limit، تحقق، خصوصية). لشهادة ISO 27001 يلزم بناء نظام إدارة أمن معلومات (ISMS) وتقييم مخاطر وضوابط إضافية لدى المشتري.

### ISO/IEC 25010 (جودة منتج البرمجيات — Software Product Quality)

| خاصية جودة | وضع المشروع | ملاحظات |
|-------------|-------------|----------|
| الوظائف (Functional suitability) | ✅ | مواصفات المنتج (PRODUCT_SPEC)، تغطية الميزات |
| الموثوقية (Reliability) | ✅ | معالجة أخطاء، health/readiness، نسخ احتياطي موثّق |
| إمكانية الاستخدام (Usability) | ✅ | واجهة عربية/إنجليزية، RTL، سمة فاتحة/داكنة، توثيق المستخدم (ADMIN_GUIDE) |
| أداء (Performance) | ✅ | Code splitting، rate limiting، تحليل الحزم (build:analyze) |
| أمان (Security) | ✅ | انظر ISO 27001 أعلاه و SECURITY.md |
| قابلية الصيانة (Maintainability) | ✅ | بنية واضحة، توثيق، MIGRATIONS، اختبارات |
| قابلية النقل (Portability) | ✅ | Docker، توثيق نشر بديل (DEPLOY_DOCKER_VPS)، Render |

**الخلاصة:** المشروع يلامس أغلب جوانب نموذج الجودة ISO 25010؛ لا يوجد تقييم رسمي موثق ضد المعيار بالكامل.

## 3. ما الذي ينقص للحصول على شهادة ISO؟

- **شهادة رسمية:** يجب التعاقد مع جهة إصدار شهادات معتمدة (accredited certification body) وإجراء التدقيق وفق معيار ISO المطلوب.
- **نظام إدارة:** لـ ISO 9001: نظام إدارة جودة كامل (سياسات، أهداف، مراجعات، سجلات). لـ ISO 27001: نظام إدارة أمن معلومات (ISMS) وتقييم مخاطر وضوابط حسب الملحق A.
- **نطاق الشهادة:** عادة تشمل المنظمة (المشتري/المؤسسة) وليس المنتج وحده؛ المنتج (UniPilot) يمكن أن يكون ضمن نطاق التدقيق.
- **وثائق إضافية:** سياسات رسمية، إجراءات موثقة، سجلات تدقيق، مراجعات إدارية (حسب المعيار).

---

## 4. توصيات للمشتري

- إذا كان الهدف **بيعاً أو نشراً بمعايير جودة عالية** دون شهادة: التوثيق والاختبار والأمان الحالي يوفّر مستوى جودة جيداً ومناسباً لعروض 10k+ كما في PREMIUM_PRODUCT_READINESS.
- إذا كان الهدف **السير نحو شهادة ISO** لاحقاً:
  - استخدام هذه الوثيقة كأساس للتوافق الحالي.
  - التعاقد مع جهة إصدار شهادات معتمدة وبدء بناء نظام الإدارة (جودة أو أمن معلومات) حسب ISO 9001 أو 27001.
  - إجراء تقييم مخاطر أمنية وتحديد فجوات إضافية.
  - الاستعانة بـ LEGAL_REVIEW_NOTES لمراجعة الشروط والخصوصية ومعالجة البيانات (خصوصاً للـ EU أو المؤسسات الكبيرة).

---

## 5. خلاصة

| سؤال | إجابة |
|------|--------|
| هل المشروع **معتمد** من ISO؟ | **لا.** لا توجد شهادة رسمية من جهة معتمدة. |
| هل المشروع **متوافق مع ممارسات** الجودة والأمان المرتبطة بـ ISO؟ | **نعم جزئياً:** أمان (OWASP، headers، تحقق، خصوصية)، توثيق، اختبارات، نسخ احتياطي، استمرارية، وثائق تجارية وقانونية. |
| هل المشروع **جاهز كأساس** للسير نحو ISO لاحقاً؟ | **نعم.** المشروع يوفّر أساساً قوياً؛ الشهادة نفسها تتطلب بناء نظام إدارة وتدقيقاً من جهة معتمدة. |
